CD&UD

Open RAN 是一种先进的网络架构方法。这一愿景的核心是两个主要组件：分布式单元 （DU） 和中央单元 （CU）。

分布式单元 （DU）

分布式单元 （DU） 处理协议栈的较低层，包括上层物理层、MAC 层和 RLC 层。其主要职责是：

      • 数据组织和管理：DU 的任务是准备要传输的数据。它对数据进行结构化，确保其采用正确的格式以实现有效的无线电传输。
      • 与无线电单元 （RU） 的交互：DU 直接管理与无线电单元的数据通信，将有组织的数据转换为无线电波进行传输，反之亦然。
      • 减少延迟和提高效率： DU 更靠近 RU 并管理较低的协议层，可确保最小的延迟，这对于实时数据交换应用程序尤其重要。

打个比方，把DU想象成交响乐的熟练技术人员，在演奏前将乐器调到完美。

名词解释：

  • *上层物理层：这是物理层中最靠近MAC层的部分。它涉及调制、编码和其他为无线电波传输准备数据所必需的过程等方面。*
  • *MAC（介质访问控制）层：该层负责数据包在网络上的放置方式。它解决了何时可能传输数据等问题，并通过管理对物理网络介质的协议访问来帮助防止冲突。*
  • *RLC（Radio Link Control）层：该层确保用户设备和网络之间的可靠数据传输。它处理分段、数据包的重组和纠错。*（透明模式，不可确认模式，确认模式)

中央单元 （CU）

中央单元 （CU） 负责监督协议栈的较高层，尤其是 SDAP、PDCP 和 RRC 层。其主要作用包括：

      • 网络管理： CU 管理网络内数据流的更广泛方面，决定数据包如何通过基础设施。
      • 与DU的协调： 为确保核心网络与无线电单元 （RU） 之间的连贯通信，CU 与 DU 保持联系，指导他们执行数据管理任务。
      • 战略决策： 对于总体网络决策，例如用户移动性管理和建立用户特定的通信承载，CU 起着举足轻重的作用。

为了与之相提并论，CU类似于交响乐指挥，确保每个部分和谐地结合在一起，以实现完美的表演

名词解释

  • SDAP层

  • 功能：

    • 主要负责适配上层服务数据，例如QoS流到DRB（Data Radio Bearer）的映射。

      //QoS流是5G网络中定义的一种逻辑概念，它代表了一个或一组具有相同QoS需求的IP流。每个QoS流都有一个与之关联的QoS
      指标集，例如5QI（5G QoS Indicator），这决定了数据传输的优先级、延迟、吞吐量等特性。
      //DRB是在无线接口上承载用户数据的实体，它负责在空中接口上传输数据，并确保数据按照一定的服务质量要求被发送和接收。DRB可以分为GBR（Guaranteed Bit Rate）和Non-GBR两种类型，分别对应于有保证比特率和无保证比特率的服务。

    • 提供QoS流到DRB的映射，确保不同服务质量要求的数据能够正确地通过网络传输。

    • 可以标记QoS流ID，以便在下层进行区分和处理。

      为了通俗易懂地l理解QoS流和DRB的概念，我们可以将其比喻为高速公路系统中的车道管理。

      QoS流（Quality of Service Flow）

      想象一下，高速公路上有不同的车辆，它们的目的地和行驶需求各不相同。有些是运送急救物资的救护车，需要最快到达目的地；有些是载满乘客的旅游巴士，需要稳定的速度；而有些则是普通的私家车，它们的行驶需求相对灵活。

      在5G网络中，QoS流就像是这些不同类型的车辆，代表着具有特定服务质量需求的数据流。例如：

      • 救护车：类似于视频通话或在线游戏，需要低延迟和高可靠性。
      • 旅游巴士：类似于高清视频流，需要稳定的带宽和良好的传输质量。
      • 私家车：类似于网页浏览或文件下载，它们的需求较为灵活，可以容忍一定程度的延迟或丢包。

      DRB（Data Radio Bearer）

      在我们的高速公路比喻中，DRB就像是不同的车道，它们为不同的车辆提供服务。车道可能有以下几种：

      • 应急车道：专门用于救护车，确保它们能以最快速度通行。
      • 客车专用道：为旅游巴士预留，确保乘客的舒适和安全。
      • 普通车道：供大多数私家车使用，根据交通状况调整速度。

      在5G网络中，DRB是无线接口上的承载实体，负责根据QoS流的需求分配无线资源。例如：

      • GBR（Guaranteed Bit Rate）DRB：为需要保证比特率的服务预留专用资源，如视频通话。
      • Non-GBR DRB：为那些不需要固定比特率的服务提供更灵活的资源分配，如文件下载。

      映射关系

      当车辆（QoS流）进入高速公路（5G网络）时，它们会被分配到适当的车道（DRB）。这样，救护车（视频通话）会直接进入应急车道（GBR DRB），而私家车（文件下载）则可能在普通车道（Non-GBR DRB）中根据交通情况行驶。

      通过这种方式，QoS流和DRB的映射确保了网络资源的有效利用，同时也满足了不同数据流的服务质量需求。

  • PDCP层

  • 功能：

    • 负责对IP数据包进行压缩和解压缩，减少无线接口上的开销。
    • 实现数据的加密和完整性保护，保证数据安全。
    • 执行序号管理和重复数据包检测，确保数据的有序性和避免重复。
    • 进行RoHC（Robust Header Compression）头压缩，提高传输效率。

  • RRC层

  • 功能：

    • 负责无线资源的管理和控制，包括建立、修改和释放无线承载。
    • 执行UE（User Equipment）和网络之间的连接管理，如建立、维持和释放连接。
    • 控制UE的移动性管理，如小区选择、重选和切换。
    • 配置UE的测量，用于网络优化和切换决策。
    • 发送系统信息，如小区接入参数、频率信息等。

移动通信协议L1/L2/L3

三层指

L1(物理层)

PHY (Physical Layer，物理层)

L2 (数据链路层) ，又称无线网络层，实现了 LTE 的无线通信功能.

PDCP(Packet Data Convergence Control，分组数据汇聚控制层)
RLC (Radio Link Control，无线链路控制层)
MAC (Medium Access Control，媒体访问控制层)

L3(网络层)，又称传输网络层，采用 传输技术对用户面和控制面数据进行传输.

RRC (Radio Resource Control，无线资源控制层)
NAS(Non Access Stratum，非接入层协议)
